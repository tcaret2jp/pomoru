# 開発ロードマップ

`requirements.md` および `design.md` に基づいた開発計画。

## Phase 1: MVP (Minimum Viable Product) - コア機能の実装
「シンプルかつモダン」なポモドーロタイマーとして、最低限動作する状態を目指す。

### 1.1 プロジェクト基盤構築
- [ ] Next.js (App Router) プロジェクトの初期化
- [ ] TypeScript, Tailwind CSS の設定確認
- [ ] ESLint, Prettier のルール整備
- [ ] 必要なアセット（アイコン、サウンド等）の準備（またはプレースホルダー）

### 1.2 UI コンポーネント作成 (静的)
- [ ] **Layout**: ヘッダー、フッターを含まないシンプルなメインレイアウト
- [ ] **TimerDisplay**: 時間表示部分（JetBrains Mono等を使用）
- [ ] **ModeSwitcher**: Work / Short Break / Long Break の切り替えタブ
- [ ] **ControlButtons**: Start / Pause / Reset ボタン

### 1.3 タイマーロジック実装
- [ ] カスタムフック `useTimer` の設計と実装
  - `setInterval` または `requestAnimationFrame` を用いた正確なカウントダウン
- [ ] Start / Pause / Reset の状態管理
- [ ] モード切り替えによる時間の変更（25分 / 5分 / 15分）

## Phase 2: 設定機能とユーザビリティ
ユーザーが自分好みに調整できる機能を追加し、実用性を高める。

### 2.1 設定モーダル実装
- [ ] **SettingsModal** コンポーネント作成
- [ ] 時間設定フォーム（各モードの分数を入力可能に）
- [ ] 自動開始（Auto-start）のトグルスイッチ実装

### 2.2 データ永続化
- [ ] `localStorage` を使用して設定内容を保存
- [ ] リロード時に設定を復元するロジック実装

### 2.3 通知機能
- [ ] タイマー終了時のアラート音再生
- [ ] ブラウザ通知 (Web Notification API) の実装と権限リクエスト処理

## Phase 3: デザイン・UX磨き込み
「静寂と集中」を体現するビジュアルとインタラクションを実現する。

### 3.1 スタイリング詳細化
- [ ] `design.md` に基づくカラーパレットの適用（作業中・休憩中でアクセントカラー変更）
- [ ] フォント（Inter, JetBrains Mono）の適用
- [ ] ダークモード/ライトモードの完全対応

### 3.2 アニメーション・レスポンシブ
- [ ] タイマー進行時のプログレスバーまたはサークルアニメーション
- [ ] モード切り替え時のスムーズなトランジション
- [ ] モバイル端末（スマホ・タブレット）での表示崩れ修正

## Phase 4: 拡張機能 (Optional / Future)
`requirements.md` の「3.3. 統計・進捗」およびさらなる機能拡張。

- [ ] **統計機能**: 今日の完了ポモドーロ数のカウントと表示
- [ ] **PWA対応**: アプリとしてのインストール対応
- [ ] **タイトルバー連携**: ブラウザタブのタイトルに残り時間を表示
